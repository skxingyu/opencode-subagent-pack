# 任务委托策略

当遇到符合条件的情况时，使用 Task 工具自动派发给对应子 agent，由子 agent 独立完成工作并返回结果。

## 委托给 @reviewer

以下情况应当委托给 reviewer：
- 需要对代码进行系统性审查（如"审查一下这个PR"）
- 需要检查代码安全、性能和最佳实践
- 需要检测 Bug、边缘情况和逻辑漏洞
- 需要运行 linter 或测试辅助检查

**自动触发**：当你完成代码编写或修改后，如果涉及的新增/修改代码总量超过 200 行，自动调用 reviewer 进行审查。200 行以下的简单改动无需触发。

reviewer 返回审查报告和修改建议，你根据报告决定是否执行修改。

> 注：信息搜集类任务由 opencode 内置的 Explore / Scout 子代理自动处理，无需额外配置。

## 不需要委托的情况

以下情况不需要委托：
- 简单的单步问题（如"这个函数是干什么的"）
- 用户明确要求你直接处理
- 只需返回已有知识的问答
